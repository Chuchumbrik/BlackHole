# Дорожная карта до MVP (фазы)

Связано: [[01 — Обзор проекта и цели MVP]], [[10 — Версионирование релизов]], [[07 — Баланс и формулы (v1)]]

## Связь версии и фаз

Формат до релиза MVP: **`0.ФАЗА.ПАТЧ`**

| Число | Роль |
| ----- | ---- |
| **0** | Пре-MVP ветка разработки. |
| **ФАЗА** (minor) | Номер **фазы** дорожной карты ниже; повышается только при выполнении критериев выхода из фазы. |
| **ПАТЧ** | Итерации **внутри фазы**: PR, исправления, мелкие дополнения без перехода к следующей фазе. |

Переход **на `1.0.0`** — см. [[10 — Версионирование релизов]] (релиз MVP по Must Have).

Текущее состояние (обновлять при смене фазы / патче): **`0.2.0`** — **фаза 2** (четыре улучшения дыры, UI «Улучшения», связка с симуляцией). Далее по плану — **0.3.x** (планета). Журнал релизов: `CHANGELOG.md`.

---

## Фаза 0 — Инфраструктура **`0.0.x`**

**Цель:** репозиторий собирается, деплоится, документация подключена.

**Состав:** Vite + React + TS + PixiJS v8 + Zustand + react-i18next; canvas + UI-оверлей; Vercel; заглушка сцены.

**Критерий выхода:** CI/ручная сборка успешна, превью на Vercel открывается, версия в UI совпадает с `package.json`.

---

## Фаза 1 — Ядро поля и экономика объектов **`0.1.x`**

**Цель:** замкнутый **core loop** без апгрейдов полной прокачки (можно временные константы ради теста).

**Must deliver:**

- Спавн с базовой частотой и **весами типов** (55 / 25 / 15 / 5), MP из диапазонов **равномерное целое** ([[07 — Баланс и формулы (v1)]]).
- Движение объектов, **радиус притяжения** и **горизонт** как игровые параметры (хотя бы константы).
- Контакт с горизонтом → поглощение (анимация/частицы минимум заглушка), начисление **MP**.
- Игровое время для RNG интервалов **не завязано на FPS** ([[09 — Вопросы и решения (журнал)]]).
- Единый стор симуляции (Zustand) и возможность стабильно «крутить» тик.

**Критерий выхода:** можно удерживать экран и видеть стабильный поток объектов и рост MP по правилам таблицы типов.

**Ссылки по дизайну:** [[02 — Core loop и улучшения дыры]], [[07 — Баланс и формулы (v1)]].

**Код (фаза 1):** `src/game/` (баланс, RNG, шаг симуляции), `GameCanvas.tsx` (Pixi + `requestAnimationFrame`, накопитель спавна по `dt`).

## Фаза 2 — Четыре улучшения чёрной дыры **`0.2.x`**

**Цель:** все **четыре ветки** из ТЗ с ценами и эффектами.

**Must deliver:**

- Цена: первый уровень «Размер» **10 MP**, далее **×1.45** по ветке.
- Эффекты за уровень: **+12%** горизонт, **+18%** гравитация, **+8%** MP диск (разблокировка при сумме уровней **≥ 5**), **+7%** MP и **+4%** скорость поглощения (разблок при сумме **≥ 10**).
- UI вкладки «Улучшения», покупка уровней, отображение эффектов/стоимости.

**Критерий выхода:** прокачка реально меняет радиусы/доход и блокировки диска/эффективности соблюдены.

**Код (фаза 2):** `src/game/upgrades.ts` (цены, разблокировки, множители), расширение `useGameStore`, `UpgradesPanel.tsx`, доработка `simulation.ts` (`gravityAccel` в `SimLayout`), `GameCanvas.tsx` (layout из уровней, MP при поглощении, кольцо диска).

## Фаза 3 — Планета **`0.3.x`**

**Цель:** один актив «планета» по ТЗ.

**Must deliver:**

- Разблокировка **750 MP** одноразово; стадии и **базовые длительности**; ускорение **100 MP = 1 мин** прогресса.
- События: интервал **6–12 мин** (uniform), таблица исходов и эффектов на «производство» планеты (**и частота объектов с планеты, и MP с них**).
- Бусты с продвинутых объектов: **45 с**, не стакаются; счётчик **8–12 объектов** до буста (uniform).

**Критерий выхода:** планета развивается, события и бусты влияют на игру предсказуемо по формулам.

**Ссылка:** [[04 — Планета — стадии, события, объекты]].

## Фаза 4 — Prestige **`0.4.x`**

**Цель:** ребёрт и постоянные бонусы.

**Must deliver:**

- Доступ при **≥ 10 000 MP** текущих; начисление **PP = floor(sqrt(MP/1000))**.
- Сброс: дыра + MP; **планета сохраняется**; сброс активных бустов и логики событий по журналу ([[09 — Вопросы и решения (журнал)]], [[03 — Prestige и прогрессия]]).
- Дерево **5 перков**, линейная разблокировка; цены перка 1 с **`Math.ceil(price × 1.8)`**, остальные по таблице.

**Критерий выхода:** полный цикл «накопил → сжал → купил перк → следующий забег».

## Фаза 5 — Оффлайн и сохранения **`0.5.x`**

**Цель:** прогресс не теряется; оффлайн честный.

**Must deliver:**

- **localStorage** + автосохранение **каждые 30 с**; при необходимости заготовка под IndexedDB.
- Оффлайн: **75%**, кап **12 ч**, скорость из **пересчёта последнего сохранённого состояния** (не снимок вкладки).
- Поле **`lastSaveTimestamp` (ISO)** для последующего лидерборда.

**Критерий выхода:** закрыл вкладку → вернулся → оффлайн MP начислен по правилам без эксплоита.

**Ссылка:** [[06 — Технические требования]].

## Фаза 6 — Полный UI MVP и статистика **`0.6.x`**

**Цель:** все экраны из ТЗ «живые», тексты на **RU** через i18n.

**Must deliver:**

- Вкладки: Космос, Улучшения, Планета, Престиж, Статистика — без «заглушка навсегда» для MVP-функций.
- Статистика: минимум «всего поглощено», «время в игре» и др. по договорённости с ТЗ.
- EN можно подключить позже; ключи i18n готовы к расширению.

**Критерий выхода:** новый игрок проходит все разделы без мёртвых кнопок Must Have.

**Ссылка:** [[05 — Визуал и UI экраны]].

## Фаза 7 — Лидерборд **`0.7.x`**

**Цель:** локальный хайскор + фейковый топ.

**Must deliver:**

- Лучший локальный результат.
- Топ-100 процедурный: масса **500–250 000** (лог-распределение), имена sci-fi + **30%** реальных; **seed = lastSaveTimestamp**; стабильность до нового сохранения.

**Критерий выхода:** таблица воспроизводима и не ломается при сохранении.

## Фаза 8 — Визуал, производительность, мобильный **`0.8.x`**

**Цель:** соответствие разделу визуала и перф-требованиям ТЗ.

**Must deliver:**

- Аккреционный диск / лёнз-ощущение по мере прокачки; поглощение заметнее (растяжение/particular).
- Ограничение **200–300 объектов**, цель **60 FPS** на типичном мобильном.
- **Responsive + touch** для основных действий.

**Критерий выхода:** игра комфортна на узком экране и не просаживается при полном спавне по настройкам.

## Порядок работ и параллельность

- Фазы **строго по зависимостям**: 1 → 2 → … по номеру; часть UI (**6**) может начинаться неполной параллельно с **5**, но минор не повышают, пока не закрыт критерий текущей фазы.
- Оценка ТЗ: **2–4 недели** на MVP — предполагается плотная работа и параллель «код + баланс», иначе урезать объём только по согласованию с заказчиком (не silent scope creep).
