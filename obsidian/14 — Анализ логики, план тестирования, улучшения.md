## 2. План тестирования (экран → функциональность → тип теста)

### Космос (canvas)
- Спавн/захват обломков → объекты влетают и поглощаются: **браузер/смоук** + **юнит** на `randomBoundaryVelocity` направленность.
- Орбиты планет стабильны на ×1, не разлетаются → **браузер** (наблюдение) + **юнит** на `integratePlanetBodies` (энергия орбиты за N шагов).
- Совпадение пунктира-предпросмотра с фактической траекторией → **браузер** на ×1 и ×10 (см. находку CRIT-1/3).
- Рост дыры дестабилизирует орбиты, планеты разрушаются → **сценарий** (прокачать гравитацию/массу, наблюдать).
- Дань: цивилизованная планета шлёт корабли, часть захватывается → **сценарий** (довести до civ≥1, мерить долю долетевших — **инструментованный прогон**).

### Улучшения
- Покупка каждой ветки даёт эффект; анлоки по сумме → **юнит** (`canPurchaseUpgrade`, `levelSum`) + **браузер**.
- Оптовая ×1/2/5/10: берёт максимум по балансу, не переплачивает/не обходит потолок → **юнит** на `buyUpgrade(count)`.
- Особые MP-апгрейды множат доход/спавн/хокинг → **юнит** на `mpUpgradeModifiers` + **браузер**.

### Планета
- Терраформинг двигает 6 параметров к 50, списывает MP → **юнит** + **браузер** (проверено).
- Щит даёт защиту от откатов на время → **юнит** на `damagePlanet` под щитом.
- Ускорение стадии за MP → **юнит** `accelerationCostMp`; **ВАЖНО:** проверить, продвигает ли жизнь/цивилизацию (находка C2b — НЕ продвигает).
- Откат при ударе астероида → **сценарий** (дождь астероидов на живую планету).

### Престиж
- `PP=floor(sqrt(MP/1000))`, порог 1000, коллапс сбрасывает ран и сохраняет мету → **юнит** `ppFromMass` + **браузер** (проверено).
- Перки B (доход/хокинг) и A (спавн/планеты/стартмасса) применяются → **юнит** `prestigeModifiers`/`prestigeRunStart`.

### Статистика
- Достижения открываются по порогам, дают бонус, переживают сжатие → **юнит** `newlyUnlocked`/`achievementMpMul`; **ВАЖНО:** `first_prestige` недостижимо при трате PP (находка C1).

### Кросс-функционально
- Сейв переживает перезагрузку → **браузер** (проверено).
- Оффлайн-доход (75%/12ч) + окно → **браузер** (проверено) + **юнит** на формулу.
- **Миграция версии схемы сейва** → **юнит/ручной** (находка C1-econ: молчаливый wipe).

## 3. Находки аудита (по severity)

### Критические
- **EC-1 (сейв):** при несовпадении `SAVE_SCHEMA_VERSION` `loadSave` молча возвращает `null` → стор стартует как «новый игрок», **стирая постоянную мету** (PP/перки/достижения). Любой будущий bump версии = wipe у всех. `saveGame.ts:48-51`. → Бэкап старого сейва + миграции + частичная загрузка постоянных полей.
- **PH-1 (физика):** объекты интегрируются **одним шагом Эйлера** на `simDt` до 1.2с (×10), без суб-шагов (у планет — есть). Туннелирование сквозь горизонт/поверхность, накачка энергии орбит, расхождение с предпросмотром. `simulation.ts` (stepSimulation/advanceObjectOneStep). → Суб-шаги с потолком h≤0.05с + свип-проверка захвата.
- **PH-2 (физика/баланс):** `effRatio` (эффективность) масштабирует ВСЮ дельту скорости (гравитация дыры+звезды+планет), а тяга корабля добавляется уже после → искажение всей системной гравитации и баланса кораблей. `simulation.ts:421-432`.
- **PR-1 (достижения):** `first_prestige`/`prestige_25` проверяют ТЕКУЩИЙ PP; при трате PP под ноль достижение де-факто недостижимо. `achievements.ts:51,58`. → Персистентный `lifetimePp`/`prestigeCount`.
- **PL-1 (геймплей):** `acceleratePlanet` двигает только геологические стадии, НО не `tickPlanetLife` → платное ускорение НЕ ускоряет жизнь/цивилизацию (разрыв обещания кнопки и петли «жизнь→дань»). `store acceleratePlanet`.

### Важные (major)
- **PH-3:** рассинхрон позиций планет (тела-ref vs стор): реконсиляция только в одну сторону (`length>nPl0`), геометрия снапшота зависит от `planetIndex` → при промахе `posById` планета «телепортируется». Ховер/клик сразу после `setActiveSystem` бьёт по старым координатам.
- **PH-4 / EC-perf:** потолок `MAX_OBJECTS_ON_FIELD` обходится `concat` дани и debris-burst (только `trySpawn` его уважает). На ×10 с цивилизациями поле пухнет >260, O(n²) в столкновениях/предпросмотре. + `tributeAccum` Map не чистится при удалении планеты (утечка).
- **EV-1:** длительности событий в игровых секундах → на ×10 событие длится 2-3 реальных секунды (баннер мелькает). → Привязать к реальному времени или клампить.
- **PL-2:** фоновые системы: цивилизации растут и истощают `mpYieldMult`, но дани не дают (дань только для активной системы) → «тихое выжимание» планет без выгоды.
- **EC-1b (оффлайн-семантика):** EMA в MP/игр-сек × РЕАЛЬНЫЕ секунды; активные игроки на ×10 систематически недополучают оффлайн. → Решить семантику явно.
- **EC-bal:** доход = произведение 5 экспоненциальных каналов (disk×eff×perk×mpUpg×ach×event), а цены растут только ×1.53 → гиперинфляция на дистанции. → Diminishing/мягкий кап.

### Минорные (выборка)
- Гонка тостов достижений (несколько за тик → виден только последний). `store unlockAchievement`.
- `lifeEmergenceRatio` показывает 0 при временной нестабильности (накопленный прогресс «скрыт»).
- Цивилизация растёт независимо от стабильности экосистемы (нельзя «погубить» зрелую цив дестабилизацией) — снимает агентность.
- Терраформинг лечит все 6 параметров за фикс. 120 MP независимо от числа дефицитов — дёшево.
- `VELOCITY_DAMPING=1` мёртвый код; `runStart.extraPlanets` — мёртвое чтение в тике.
- `pickEvent`/мир несидированы (невоспроизводимо — важно для будущего лидерборда).
- Хокинг-пассив множится только на perk/mpUpgrade hawkingMul, НО не на achMul/eventMpMul — уточнить намеренность.

## 4. Улучшения (логика + геймдизайн), приоритезированно

**Технический долг (до релиза):**
1. Миграции сейва + бэкап (EC-1) — критично, иначе релиз = риск wipe.
2. Суб-шаги интегратора объектов + свип-захват (PH-1) — честность орбит/предпросмотра.
3. Единый источник позиции планеты + двусторонняя реконсиляция (PH-3).
4. Единый `addObjects` с потолком на всех путях + чистка `tributeAccum` (PH-4).
5. `effRatio` только на гравитацию дыры (PH-2).
6. `lifetimePp`/`prestigeCount` для достижений (PR-1).

**Геймдизайн (удовольствие/глубина):**
1. `acceleratePlanet` ускоряет и жизнь/цивилизацию, либо отдельный «ускоритель жизни» с понятным ROI через дань (PL-1).
2. Сгладить кривую PP (ранние сжатия дают 3-5 PP, не 1) — power-spike после первого сжатия.
3. Diminishing на произведение каналов дохода — растянуть прогрессию, продлить ценность prestige.
4. События читаемы на любой скорости (масштабировать длительность) — «момент радости».
5. Фоновые цивилизации копят «накопленную дань» к возврату игрока (приятный сюрприз) вместо тихой деградации.
6. Очередь тостов достижений; показывать прогресс жизни даже при временной нестабильности; дать возможность погубить/спасти зрелую цивилизацию (агентность над зрелыми планетами).
7. Сидированный RNG мира — задел под воспроизводимость/лидерборд.
