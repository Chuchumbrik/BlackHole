# План реализации MVP (играбельный)

Связано: [[01 — Обзор проекта и цели MVP]], [[11 — Дорожная карта до MVP (фазы)]], [[03 — Prestige и прогрессия]], [[07 — Баланс и формулы (v1)]]

> Рабочий документ-спецификация на текущий цикл. Источник истины по фактам кода — `CLAUDE.md` в корне репо. Составлен 2026-06-16.

## 0. Цель и принципы

**Цель:** насыщенный **играбельный MVP** (не минимальный): замкнутая петля удержания + содержание (разнообразные прокачки, достижения, физичная симуляция, развитие планет), затем визуальный полиш.

**Зафиксированные решения:**
- Сейв — **локальный** (`localStorage`), без облака/аккаунтов.
- Оффлайн-доход — от **усреднённого дохода MP/с** (EMA), **75%**, кап **12 ч**.
- Prestige — **data-driven магазин перков**, две валюты (MP внутри рана, PP между ранами).
- Физика — **главные тела динамичны** (звезда + планеты + дыра взаимно гравитируют; планеты орбитируют из-за массы звезды), **обломки — пробные частицы** (подчиняются полю всех крупных тел, но не тянут друг друга).
- Визуальный полиш — **финальный этап**.

**Инженерные принципы:**
1. **Data-driven каталоги** (апгрейды / перки / достижения) — контент добавляется записью в данные, не кодом.
2. **Единый сборщик модификаторов** `computeModifiers(levels, perks, ctx)` → симуляция/генерация/доход читают только его.
3. Каждая фаза проверяется в браузере (dev + puppeteer: консоль, скриншот, «прогресс пережил перезагрузку»).
4. Ритуал версий: bump patch + `CHANGELOG.md` на каждую фазу.

**Что НЕ в этом MVP (явно):** мультизвёздные системы; события/сверхновая как отдельная механика; аккаунты/облако/синхронизация; лидерборд/античит.

## 1. Фаза A — Каркас: data-driven каталог апгрейдов

**Проблема (из аудита кода):** апгрейды захардкожены `enum`-ом; эффекты разбросаны по `upgrades.ts`, `balance/upgradeEconomy.ts`, `balance/branchA46.ts`, `balance/unlocks.ts`; UI (`UpgradesPanel`) и `GameCanvas` содержат `switch/if` по веткам. ~47 мест читают `upgradeLevels`. Добавление 8-й ветки сейчас — правка в ~8 местах.

**Делаем:**
- Тип определения: `{ id, currency: 'MP'|'PP', nameKey, baseCost, costMult, maxLevel, unlock, effects[], i18nKey }`.
- `effects[]` описывают оси воздействия (множитель дохода, радиус горизонта, радиус гравитации, ускорение притяжения, тяга кораблей, вес редких, пассив, проки джетов, визуал).
- **Единый сборщик** `computeModifiers(...)` → один derived-объект; все потребители (`simulation.ts`, `GameCanvas`, `planet*`, `ViewScaleControls`, `UpgradesPanel`) читают его, без `switch`.
- UI `UpgradesPanel` — итерация по каталогу, без захардкоженных веток.

**Решение по миграции:** существующие 7 веток (`size, gravity, disk, efficiency, jets, lensing, hawking`) переводим в каталог в этой фазе (единообразие — на нём висят достижения, перки, оптовая покупка). Риск регресса гасим браузерным смоуком (покупка каждой ветки → эффект виден).

**Затронуто:** `src/game/upgrades.ts`, `src/game/balance/*`, `src/components/UpgradesPanel.tsx`, `src/components/GameCanvas.tsx`, `src/components/ViewScaleControls.tsx`, `src/store/useGameStore.ts`.

**Приёмка:** все 7 веток покупаются и дают эффект как раньше; добавление новой прокачки = одна запись в каталоге.

## 2. Фаза P — Физика: динамические главные тела

**Сейчас:** планеты кинематические — `angle = orbitPhaseRad + simTimeSec·orbitSpeed`, идеальная окружность по скрипту (`planetLayout.ts:48-50`). Гравитационный фундамент уже настоящий: `accel = G·m/(dist²+soft)` (`simulation.ts:198-199`), массы у звезды/планет/обломков есть; обломки уже притягиваются дырой+звездой+планетой.

**Делаем:**
- Планеты → **динамические тела**: добавить `vx, vy`; стартовая скорость = круговая орбита из массы звезды (vis-viva: `v = sqrt(G·M_star / r)`, перпендикулярно радиусу).
- **Взаимная гравитация главных тел** (звезда ↔ планеты ↔ дыра): интегрировать каждый шаг. Тел мало → O(k²) дёшево.
- **Обломки/корабли** — пробные частицы: суммарное поле от всех крупных тел (звезда + каждая планета + дыра). Расширить существующее (сейчас учитывается звезда+дыра+планета-владелец SOI) до всех планет.
- **Рост дыры влияет на орбиты:** по мере роста массы дыра возмущает и в пределе разрушает орбиты — эмержентный «разрушить систему» (совпадает с видением [[01]]).
- **Столкновения планет (Q-P1, решено):** возможны, но **только как следствие нарушения баланса игроком** (рост массы дыры возмущает орбиты → сближение/слияние). **Начальная генерация обязана давать стабильные непересекающиеся орбиты** — спонтанных столкновений «из коробки» быть не должно (разнос орбит по радиусу + круговые стартовые скорости).
- **Поглощать можно всё:** дыра поглощает обломки/корабли/планеты (есть) + **звезду**. Поглощение звезды — НЕ «конец», а веха масштаба (см. Q-P2 / §14 «Прогрессия масштаба»).
- **Стабильность:** полу-неявный Эйлер / velocity-Verlet; **суб-шаги при ускорении времени** (×10) — фиксированный `dtPhysics`, дробим `simDt`, чтобы орбиты не «разлетались».

**Влияние на сейв:** хранить **позиции и скорости** всех тел (а не `orbitPhase`). Схема сейва должна это учесть.

**Затронуто:** `src/game/simulation.ts`, `src/game/world/types.ts`, `src/game/world/generation.ts` (старт. скорости), `src/game/world/planetLayout.ts` (перевод с кинематики на чтение позиции тела), `src/components/GameCanvas.tsx`.

**Приёмка:** при генерации планета сама встаёт на стабильную орбиту из массы звезды; рост дыры заметно искажает орбиты; FPS в бюджете (см. Фаза G).

## 3. Фаза S — Сейв + оффлайн

**Сейв:**
- Ключ `localStorage["cbh:save"]`, обёртка `{ schemaVersion, data }`.
- Сохраняем: `massMp, gameTimeSec, upgradeLevels`, **полный `systems`** (несидированный мир) **+ позиции/скорости тел**, `viewTier, simTimeScale, activeSystemId, activePlanetId, jetBuffEndsAtSimSec`, `prestigePoints, perkLevels`, прогресс достижений, EMA-ставка, `lastSaveEpochMs`. Транзиентное (`mpGainFloaters`) — не пишем.
- Автосейв: каждые **10 c** + на `visibilitychange`/`beforeunload`.
- Битый/старый сейв: миграция по `schemaVersion`; при неудаче — мягкий сброс с уведомлением.
- Кнопка **«Сбросить прогресс»** (с подтверждением).

**Оффлайн:**
- EMA общего дохода MP/с (поглощение + хокинг) по окну ~60 c активной игры (единственный путь дохода — `addMassMp`, удобно перехватить).
- При заходе: `оффлайн = ставка · min(прошло, 12ч) · 0.75`; окно «пока вас не было: +N MP». Порог показа — оффлайн ≥ 60 c.
- **Мир за оффлайн не двигаем** (планеты «замирают»), начисляем только MP. *(на подтверждении)*

**Затронуто:** `src/store/useGameStore.ts` (сериализация/гидрация, reset), новый модуль `src/game/save/*`, `src/components/GameCanvas.tsx` (хук автосейва + перехват дохода для EMA), новая модалка оффлайна.

**Приёмка:** перезагрузка вкладки сохраняет прогресс; возврат через время даёт оффлайн-доход с окном.

## 4. Фаза R — Prestige-магазин

- Валюта **PP**; формула `PP = floor(sqrt(MP/1000))`, `totalPP` накопительно. Порог входа — привязать к балансу (предв. 10000 MP).
- **Коллапс:** `massMp→0`, `upgradeLevels→0`, jet-buff сброшен, **мир регенерируется**, `gameTimeSec→0`. Сохраняются `prestigePoints, perkLevels`, достижения, lifetime-статы. (Наследие мира — пост-MVP.)
- Перки два типа:
  - **A — старт рана** (применяются при генерации/спавне): плотность поля (больше астероидов), богатые недра (веса спавна), больше планет, больше систем, стартовая масса, гравитация звезды.
  - **B — постоянные** (в сборщике модификаторов): глобальный ×MP, буст хокинга, дешёвая сингуляр* (множитель цен), тёплый старт, оффлайн-перки.
- Цена уровня перка: `ceil(base · 1.9^level)`, у каждого свой `maxLevel`.
- Перки типа A действуют **со следующего рана**.

**Затронуто:** стор (PP/perkLevels/prestige-экшен), каталог перков (Фаза A), `generation.ts`/`spawn.ts` (чтение перков A), сборщик модификаторов (перки B), вкладка «Престиж» (Фаза UI).

**Приёмка:** при ≥ порога — «Сжать» даёт PP, сбрасывает ран, перки видимо ускоряют следующий заход.

## 5. Фаза U — Расширенные MP-апгрейды (волна 1)

**Q-U1 (решено): включаем все наработки сразу.** Данными поверх каркаса:
- **Спагеттификация** — бонус MP за крупные тела.
- **Тёмное гало** — расширяет зону гравитации без роста горизонта.
- **Магнитная воронка** — сильнее тянет корабли/металл (веса).
- **Приливный разрыв** — крупные тела рвутся на осколки у горизонта.
- **Вращение (Керр)** — бонус к захвату по касательным.
- **Горизонт событий ↑** — замедляет «побег» тел за `R_out`.
- **Ветка B «Окружение» (risk-reward)** — выше доход ценой шанса потери объекта.

## 14. Прогрессия масштаба (Q-P2 — на проработке)

Цель: поглощение звезды — не «game over», а **веха роста и переход на следующий уровень масштаба**. Опора на уже существующее: `viewTier` 0/1/2 (у дыры → система → галактика), несколько систем в модели (`PLANET_SYSTEM_COUNT`), `setActiveSystem`, prestige как «новый космический цикл» ([[03]]).

**Решение: гибрид.** Лестница:

```
объекты → планеты → ЗВЕЗДА
   ↓ поглощение звезды = СВЕРХНОВАЯ (визуал + рывок массы дыры + лут)
СИСТЕМА истощена
   ↓ карта галактики (viewTier 2)
следующая система (тяжелее / богаче)
   ↓ ×N зачищенных систем
PRESTIGE → новая вселенная, возрождаешься крупнее (PP-перки)
```

**Механика по шагам:**
- **Сверхновая.** Поглощение звезды — событие: скачок массы/радиуса дыры, выброс лута (ценные обломки/осколки), визуальный взрыв (визуал — Фаза V; механика — здесь). После — система «мёртвая» (звезды нет, орбиты планет окончательно дестабилизированы → планеты падают/выбрасываются).
- **Переход.** Открывается **карта галактики**: выбор/переход к следующей системе (`setActiveSystem`; генерация новой системы тяжелее — больше тел/масса, выше ценность).
- **Гейт prestige.** После зачистки **N систем** (число — на баланс) prestige становится доступен/выгоден; коллапс = новая вселенная с `totalPP`.

**Следствия для плана:**
- Фаза **P (физика):** добавить переход «звезда поглощена → орбиты планет без центрального тела» (планеты переходят на гравитацию дыры/выброс).
- Фаза **R (prestige):** prestige-гейт завязать на счётчик зачищенных систем, а не только на порог MP.
- Новый под-блок в Фазе **PL/мир:** генерация **следующей системы** с растущей сложностью; счётчик прогресса по галактике.
- Фаза **UI:** карта галактики как игровой экран (не только режим обзора), индикатор «истощено N систем».
- Фаза **V:** анимация сверхновой и коллапса.
- **Достижения** (ACH) естественно ложатся сюда: «первая сверхновая», «N систем», «первая звезда» и т.д.

*Примечание по масштабу: это самая «контентная» добавка; мультизвёздные системы по-прежнему вне MVP — система = одна звезда, но систем много.*
