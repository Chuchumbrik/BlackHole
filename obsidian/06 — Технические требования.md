# Технические требования

Связано: [[00 — Главная (MOC)]], [[08 — Деплой — GitHub и Vercel]], [[09 — Вопросы и решения (журнал)]]

## Стек (финальная фиксация)

- **React** + **Vite** + **TypeScript**
- Рендер: **PixiJS v8**
- Состояние: **Zustand**
- Локализация: **react-i18next** (RU в релизе, EN подключается быстро)
- Сохранения: **localStorage**; при необходимости больших данных — **IndexedDB**

Альтернатива из исходного ТЗ по рендеру: Phaser 3 — не выбрана.

## Архитектура UI и игры

- **Один canvas PixiJS** — фон, игровая сцена, частицы, объекты.
- **React** — HTML-оверлей поверх canvas: вкладки, кнопки, модалки, статистика, локализация.

Подробнее — [[09 — Вопросы и решения (журнал)]]. Линтеры, CI, Vercel PR-preview — в **README** репозитория, не в геймдизайне.

## Сохранения

- **localStorage**
- Автобэкап состояния **каждые 30 секунд**

## Оффлайн-прогресс

- Время через `Date.now()`
- **Скорость** для расчёта не «снимок при выходе», а **пересчёт по сохранённому состоянию** на момент последнего сохранения (апгрейды дыры, перки prestige, стадия планеты и т.д.) — анти-эксплойт.
- Коэффициент: **75 %**:  
  `offlineMP = onlineRateFromSavedState * secondsOffline * 0.75`
- Максимум оффлайна: **12 часов** (расширение позже, в т.ч. через prestige)

## Лидерборд (MVP)

- Локально: **лучший результат** игрока
- **Топ-100**: процедурная генерация; масса **500 … 250 000 MP** (логарифмическое распределение); имена sci-fi + **30 %** «реальных»
- **Seed**: строка **`lastSaveTimestamp`** (ISO времени последнего сохранения); до следующего сохранения список воспроизводим в сессии
