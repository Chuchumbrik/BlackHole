# Визуал и UI экраны

Связано: [[00 — Главная (MOC)]], [[02 — Core loop и улучшения дыры]]

## Визуальный стиль

- Минималистичный тёмный космос + parallax звёзды.
- Чёрная дыра с эффектом линзирования и растущим аккреционным диском.
- Реал-тайм траектории и орбиты как главный «живой» индикатор прогресса.

## Масштаб сцены (v1.5)

- На основном экране читается **иерархия масштабов**: звезда и орбитальная зона системы **велики**; дыра **мала** относительно системы на ранних этапах и **растёт** по мере прокачки — игрок визуально ощущает путь «с окраины к доминированию в системе».
- **Реализация (0.2.5):** по умолчанию в центре экрана — **чёрная дыра** (pivot камеры); центр системы (звезда) смещён в мировых координатах. **Перетаскивание** — панорама по звёздной системе (в клипе); **колёсико** — пользовательский зум поверх масштаба апгрейдов/режима вида.
- Объекты при подходе к горизонту **визуально уменьшаются** перед исчезновением (минимальный масштаб у горизонта ослаблен — см. \`SHRINK_MIN_AT_HORIZON\` в \`GameCanvas.tsx\`, релиз **0.3.1**).
- **Релиз 0.3.3:** орбита и \(R_{\mathrm{out}}\) — **`STELLAR_SYSTEM_RADIUS_MUL` = 6.875**; пределы пользовательского зума **`USER_ZOOM_MIN` / `USER_ZOOM_MAX`** (\`layoutViewport.ts\`) — сильное отдаление для обзора поля; панорама до **~3.5·min** стороны окна; подписи имён объектов компенсируют масштаб слоя (\`1/worldScale\`).
- **Релиз 0.3.4:** предпросмотр траектории — лимит точек **32000** (\`simulation.ts\`), чтобы линия доходила до терминального события; панель **скорости времени** (пауза, **×1…×5**) внизу справа на вкладке «Космос» (\`TimeScaleControls\`, \`simTimeScale\` в сторе).
- **Релиз 0.4.0 (фаза 3, ветка A 4–6):** улучшения **джеты / линзирование / хокинг** — бафф MP поля при джетах, пассивный хокинг, редкий спавн; визуал кольца линзы, лучей джетов и модуляции диска при хокинге (\`paintHole\`).
- Мобильный жест pinch для зума — при необходимости позже (фаза 8).

## Эффекты по событиям и апгрейдам

- Релятивистские джеты (полюсные лучи).
- Гравитационное линзирование фона/звёзд.
- Вспышка сверхновой, краткосрочная мини-дыра.
- Разрушение планет при опасных конфигурациях.

## UI-структура MVP

- Космос (основной экран симуляции).
- Системы/Планета (переключение систем, прогресс планет).
- Prestige (сжатие, PP, ветки перков).
- Статистика.

### Вкладка «Улучшения» (экран)

- Карточки четырёх веток (фаза 2): уровень, текст эффекта, **блок «Сейчас»** — накопленные множители к базе и для горизонта/зоны притяжения приблизительный радиус в px для текущего размера окна (\`upgradeBranchSnapshot\`, \`computeRadiiPx\` — см. [[07]]).
- Колесо мыши на основном экране: зум сцены (**на себя** — приближение, **от себя** — отдаление).

### Инспектор чёрной дыры

- Отдельная панель или секция UI: **все текущие характеристики дыры** в одном месте.
- У каждого параметра: название, **числовое значение**, иконка «i» или длинное нажатие → **подсказка** + **формула / источник** (отсылка к [[07 — Баланс и формулы (v1)]] или к имени функции в коде).
- По возможности: переключатель «только итог» / «разложение по вкладу апгрейдов».

### Улучшения — единый drawer справа

- Не отдельная вкладка-страница: **модальное окно / drawer с правого края** поверх игры.
- Внутри — разделы **«Чёрная дыра»** (ветки сингулярности и позже окружения) и **«Планета»** (стадии, ускорение, связанные апгрейды).
- Одна точка входа (кнопка на HUD / жест), закрытие не должно ломать симуляцию.

## Траектории (предпросмотр орбиты)

- Линия — **пунктир**: приглушённый для выбранного объекта, **ярче (белый) при наведении**; шаг пунктира в **мировых** координатах масштабируется обратно комбинированному зуму (\`combinedWorldScale × userZoom\`), чтобы на экране штрихи оставались пропорциональными при панораме и колёсике (\`GameCanvas\`, \`strokeDashedPolyline\`).
- Траектория рисуется для **любого объекта** при **наведении** или при **выборе кликом** (повторный клик по объекту или клик по пустому месту снимает выбор).
- При **наведении** (если объект не совпадает с текущим выбором) над объектом показывается та же строка **тип · позывной**, что и при выборе (\`displayName\`).
- Для **выбранного** объекта над спрайтом показывается подпись: **тип (RU) и программный позывной** (\`SimObject.displayName\`, генерация в \`objectNames.ts\`).
- Предпросмотр строится численно до **терминального события** (горизонт, звезда, уход за \`systemRadius\`, для корабля — правила побега), при необходимости — до **столкновения** с другим телом (\`othersSnapshot\` в \`predictTrajectoryPoints\`: те же радиусы, что в \`resolveBodyCollisions\`; позиции других тел — **снимок на кадр**, они не двигаются в расчёте предпросмотра), либо до лимита времени/точек при условной замкнутой орбите.
- На уровне «галактика» траектории и подпись скрыты.

## UX-принципы

- Игрок должен видеть прямую связь: «купил апгрейд -> изменилось движение мира».
- Риски должны быть читаемы заранее (подсветка опасных орбит, предупреждения).
- Важные события не должны теряться на мобильном экране.
- Прозрачность чисел: инспектор и формулы снижают ощущение «магических процентов».
