# Версионирование релизов (SemVer)

Связано: [[00 — Главная (MOC)]], [[01 — Обзор проекта и цели MVP]], [[11 — Дорожная карта до MVP (фазы)]]

## Источник правды

Версия продукта в репозитории: поле **`version`** в `package.json` (тот же номер дублируется в UI и в релизных заметках).

## Правила (договорённость)

### До MVP (`0.y.z`)

Формат: **`0.ФАЗА.ПАТЧ`**

| Часть | Смысл |
| ----- | ----- |
| **0** | Пре-MVP; пока не выполнен scope [[01 — Обзор проекта и цели MVP]]. |
| **ФАЗА** (второе число, **minor**) | Номер **фазы** дорожной карты — см. [[11 — Дорожная карта до MVP (фазы)]]. Повышается при выполнении критериев выхода из фазы. |
| **ПАТЧ** (третье число) | Итерации **внутри одной фазы**: PR, фиксы, мелкие дополнения без перехода к следующей фазе. |

Примеры: **`0.0.0`** — каркас после фазы 0; **`0.3.7`** — фаза 3, седьмая итерация/патч внутри неё; переход к фазе 4 → **`0.4.0`**.

### Релиз MVP

| Версия   | Смысл |
| -------- | ----- |
| **1.0.0** | **Первый релиз MVP** — playable по Must Have из ТЗ (git-тег `v1.0.0`). |

После **1.0.0** — классический SemVer: **major** — ломающие изменения / крупные этапы; **minor** — новые возможности; **patch** — исправления (уже **не** привязано к «фазам» документа 0.x, если не введёте новую схему отдельным решением).

## Связь с ТЗ

ТЗ документа (`v1.0`) описывает дизайн; **1.0.0 кода** означает «реализован MVP по этому ТЗ», а не версию документа.
