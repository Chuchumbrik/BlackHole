# Версионирование релизов (SemVer)

Связано: [[00 — Главная (MOC)]], [[01 — Обзор проекта и цели MVP]]

## Источник правды

Версия продукта в репозитории: поле **`version`** в `package.json` (тот же номер дублируется в UI и в релизных заметках).

## Правила (договорённость)

| Версия   | Смысл |
| -------- | ----- |
| **0.0.0** | Старт: каркас, инфраструктура, заглушки; до выполнения scope MVP по ТЗ. |
| **0.*.*** | Пре-MVP: итерации, фичи, баланс до готовности релиза по [[01 — Обзор проекта и цели MVP]]. |
| **1.0.0** | **Первый релиз MVP** — playable по Must Have из ТЗ (можно помечать git-тегом `v1.0.0`). |

После **1.0.0** — классический SemVer: **major** — ломающие изменения / крупные этапы; **minor** — новые возможности; **patch** — исправления.

## Связь с ТЗ

ТЗ документа (`v1.0`) описывает дизайн; **1.0.0 кода** означает «реализован MVP по этому ТЗ», а не версию документа.
